# Phase 4：PM统筹计划书详细指令

## 目标
把所有竞品的分析结果汇总成一份优先级清晰、无重复、可执行的吸收总计划。

## 输入
`absorber-workspace/[每个竞品名]/analysis.md`（所有已完成的分析报告）

## 处理步骤

### Step 1：读取所有分析报告
逐一读取每个竞品的 analysis.md，提取全部「可吸收项」（P0/P1/P2）。

### Step 2：去重合并
同一个特性可能在多个竞品中都有，处理规则：

- **完全相同的特性**：合并为一项，来源标注所有提到它的竞品，实施参考选最佳版本
- **相似但实现不同的特性**：保留最优实现，附注其他竞品的变体供参考
- **互补的特性**：保持独立，但在描述中注明关联关系

去重标准：功能目的相同 = 同一特性（不管名字是否一样）

### Step 3：全局重排优先级

在单竞品分析中，每项都有P0/P1/P2评级。汇总时，综合以下因素重新排序：

**优先级提升因素：**
- 多个竞品都有该特性（说明是行业标配）
- 影响用户第一印象（onboarding、错误提示、文档）
- 当前项目在该点被竞品明显碾压

**优先级降低因素：**
- 实施难度高且收益一般
- 依赖其他未完成项（有前置依赖）
- 与项目当前阶段不匹配（如早期项目先不做国际化）

**依赖关系排序：**
有前置依赖的项目，必须排在依赖项完成之后。

### Step 4：生成计划书

使用 `templates/absorption_plan.md` 模板，填入所有内容。
每项必须包含：
- 唯一ID（PA-001, PB-001 等，P代表Priority，A/B/C对应P0/P1/P2）
- 来源竞品（可多个）
- 价值描述（一句话说清楚用户能感知到的差异）
- 实施方案（具体到涉及哪些文件/模块，不需要写代码）
- 完成标准（可验证的验收条件）
- 难度评估（低/中/高）

存入 `absorber-workspace/ABSORPTION_PLAN.md`。

### Step 5：呈现摘要给用户确认

```
📋 吸收计划书已生成

已分析 N 个竞品，共发现 X 项可吸收特性：
- P0（高价值，优先做）：X 项
- P1（有价值）：X 项  
- P2（可选）：X 项
- 不吸收：X 项（原因见计划书）

P0 前3项预览：
1. **PA-001 [特性名]**（来自：竞品A、竞品B）
   [一句话价值描述] | 难度：低
   
2. **PA-002 [特性名]**（来自：竞品C）
   [一句话价值描述] | 难度：中
   
3. **PA-003 [特性名]**（来自：竞品A）
   [一句话价值描述] | 难度：低

完整计划书：absorber-workspace/ABSORPTION_PLAN.md

是否开始按计划执行？可以：
- 「开始执行」：从 PA-001 开始
- 「从 PA-003 开始」：跳到指定项
- 「调整优先级」：告诉我你的想法
```

等待用户确认后进入 Phase 5。

## PM角色的判断原则

作为PM，输出计划时要体现以下判断，而不只是机械汇总：

1. **战略判断**：哪些改动会让用户「哇一下」（感知明显），优先做
2. **风险意识**：高难度项目要明确标注风险，不把它们排在最前面
3. **完整性检查**：计划书里不应该有「等等看」的模糊项，每项都要可验证
4. **诚实**：没把握的实施方案要标注「需要进一步调研」，不假装确定
